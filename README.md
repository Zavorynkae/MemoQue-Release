# MemoQue 使用文档

## 一、应用简介

MemoQue 是一款基于 Android 端侧 AI 的图像分类应用。所有图像识别与数据处理均在本地完成，无需联网，保护隐私的同时提供快速响应。

**技术特性**
- 端侧 ONNX 模型推理，零网络依赖
- 自定义模板与字段，灵活适配多种场景
- 完整的导入/导出功能，数据自主可控

---

## 二、首次使用

### 2.1 启动应用

首次打开应用时，系统会自动初始化默认模板（包含名称、品牌、型号、备注等字段）。

### 2.2 必要权限

| 权限 | 用途 | 是否必须 |
|------|------|---------|
| 相机 | 拍摄物品照片进行识别 | 是 |
| 存储 | 保存记录图片、导出备份 | 是 |

---

## 三、核心功能

### 3.1 拍照登记（首页）

```
┌─────────────────┐
│   相机预览画面   │
│                 │
│    [快门按钮]    │
├─────────────────┤
│ 识别结果: 笔记本电脑 │
│ 品牌: _________ │
│ 型号: _________ │
│ 备注: _________ │
│                 │
│ [保存记录]      │
└─────────────────┘
```

**操作步骤**
1. 对准物品，点击快门拍照
2. 等待 AI 识别（或手动填写）
3. 核对/补全字段信息
4. 点击保存

---

### 3.2 记录管理

- **查看记录**：列表展示，支持按字段搜索
- **分组浏览**：按品牌/型号等字段自动分组
- **详情查看**：点击图片可全屏查看原图

---

### 3.3 设置中心

#### 模板管理
| 功能 | 说明 |
|------|------|
| 添加字段 | 自定义记录需要填写的信息项 |
| 删除字段 | 移除不需要的字段 |
| 重置模板 | 恢复默认字段配置 |

#### AI 模型设置

**加载模型**
1. 点击「选择模型」，从文件管理器中选择 `.onnx` 模型文件
2. 系统自动验证并加载
3. 加载成功后显示「✓ 模型加载成功」

**标签文件**（可选）
- 用于将模型输出的数字索引转换为可读标签
- 格式：纯文本，每行一个标签名称
- 示例：
  ```
  笔记本电脑
  手机
  平板电脑
  耳机
  ```

**模型参数配置**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 输入尺寸 | 模型接受的图像宽高 | 224 × 224 |
| 均值 (Mean) | 预处理归一化均值 | 0.485, 0.456, 0.406 |
| 标准差 (Std) | 预处理归一化标准差 | 0.229, 0.224, 0.225 |
| 通道顺序 | RGB / BGR | RGB |
| 输出类型 | Logits / Probabilities | Probabilities |
| 输入格式 | NCHW / NHWC / 自动 | 自动 |
| GPU 加速 | 使用 GPU 推理（需设备支持） | 关闭 |
| CPU 线程 | 推理使用的线程数 | 4 |

> **提示**：参数需与模型训练时的预处理一致，否则识别准确率会下降。修改后点击「保存模型参数」生效。

**输出字段映射**
- **识别结果输出字段**：选择将 AI 识别结果填入哪个字段
- **AI 分析结果字段**（可选）：额外显示置信度或原始标签

---

### 3.4 数据备份与恢复

| 功能 | 操作 | 说明 |
|------|------|------|
| 导出数据 | 点击「导出数据」→ 选择保存位置 | 生成 ZIP 备份文件，含所有记录与图片 |
| 导入数据 | 点击「导入数据」→ 选择 ZIP 文件 | 追加导入，不覆盖现有记录 |

## 四、模型要求

### 4.1 支持的模型格式
- 扩展名：`.onnx`
- 输入：单张图像（通常 3 通道 RGB/BGR）
- 输出：分类结果（Logits 或 Softmax 概率）

### 4.2 常用模型示例
- MobileNetV2
- ResNet18/50
- 自定义训练的 PyTorch/TensorFlow 模型（导出为 ONNX）

### 4.3 模型转换参考
```python
# PyTorch 转 ONNX 示例
torch.onnx.export(
    model,
    dummy_input,
    "model.onnx",
    input_names=["input"],
    output_names=["output"],
    dynamic_axes={"input": {0: "batch_size"}}
)
```

## 五、常见问题

| 问题 | 可能原因 | 解决 |
|------|---------|------|
| 模型加载失败 | 文件格式不支持 / 路径错误 | 确认是 `.onnx` 文件，重新选择 |
| 识别结果全错 | 预处理参数与模型不匹配 | 检查 mean/std、通道顺序、输入尺寸 |
| 识别速度慢 | 未开启 GPU / 线程数太低 | 尝试开启 GPU 加速，或增加 CPU 线程 |
| 标签显示为数字 | 未配置标签文件 | 在设置中上传对应的 `labels.txt` |
| 应用闪退 | 模型内存占用过大 | 尝试更小的模型（如 MobileNet） |

## 六、隐私声明

- **所有数据本地存储**：记录、图片、模型均保存在设备本地
- **无网络传输**：AI 推理不连接云端服务器
- **数据自主权**：通过导出功能可随时备份或迁移数据
